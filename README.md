# emotion-game 😄🔥😲😢

**表情だけで操作するランゲーム**です。  
カメラからプレイヤーの表情（happy / angry / surprised / sad / neutral）を読み取り、  
ジャンプ・攻撃・ブースト・ゲージ調整・メニュー操作などを全部「顔」で行います。

将来的にはスマートフォン向けに App Store での配信も視野に入れています。

---

## コンセプト

- コントローラーやタップに頼らず、**表情そのものがゲームパッドになる**
- 「笑顔でコンボをつなぐ」「怒り顔で爆弾を破壊する」など、  
  **感情表現そのものがスコア・コイン・演出に直結**する
- プレイ中に獲得したコインで **キャラの着せ替えや背景スキンをガチャで解放**できる

---

## 主な機能（現時点）

### 1. 表情認識ベースのゲームプレイ

プレイ中の主な操作はすべて表情で行います：

- 😄 **happy**  
  - ジャンプ
  - コンボ加算 & スコア加算
  - 地面にいる時のみ大ジャンプ
- 🔥 **angry**  
  - 前方に波動攻撃を発射
  - 爆弾を破壊するとスコア & コンボボーナス
- 😲 **surprised**  
  - 前進ブースト（わずかにプレイヤー位置を前方へ）
- 😢 **sad**  
  - FEVER ゲージを少しずつ減らす（ゲージ調整用）
- 😐 **neutral**  
  - 特別なアクションは無し（通常移動）

### 2. スコアリング・コンボ・FEVER

- **コンボ数**に応じてスコア倍率が上がる
- FEVER ゲージが 100 になると **FEVER TIME** に突入
  - 背景が派手なカラーに変化
  - スコア倍率アップ
  - スター出現率アップ

### 3. ミッションシステム

画面下部に「次に狙うべき目標」が常に表示されます：

- ミッション例
  - 笑顔で 10 コンボ達成
  - FEVER を 2 回発動
  - スコア 5,000 超え など

達成状況に応じてメッセージが切り替わり、  
すべてクリアすると「全ミッション達成」メッセージに変わります。

### 4. コイン & ガチャ & 着せ替え

1 プレイ終了時（GAME OVER 時）にスコアに応じて **コイン** を獲得します。

- 例：スコア 100 ごとに 1 コイン（プロトタイプ時点の仕様）
- コインはローカルストレージに保存され、次回起動時にも引き継がれます

貯めたコインを使って **ガチャ** を回すことで：

- キャラクターのスキン（色・輪郭・目・口などのテーマカラー）
- 背景スキン（グラデーション・地面・ライン色など）

を解放できます。

ガチャ結果にはレアリティがあります：

- COMMON
- RARE
- EPIC
- LEGENDARY

すでに持っているスキンを引いた場合は「既に所持」として表示されます。

### 5. キャラ / 背景の着せ替え

解放済みのスキンは、ゲーム内で装備を切り替えることで反映されます。

- キャラクター本体は `CharacterSkinColors`
- 背景は `BackgroundSkinColors`
- 使用中のスキン ID はローカルストレージに保存され、次回起動時にも反映

### 6. 表情によるコンテニュー

- ライフが 0 になり **GAME OVER** になったあと、
- 一定時間待ってから **笑顔（happy）をしばらくキープ**すると
  - ゲームがリセットされて再プレイ可能（表情コンテニュー）
- さらに、1 度だけ発動する「神回避コンテニュー」もあり：
  - ライフ 0 になるタイミングで
  - プレイヤーの表情が `happy` or `surprised` の場合
  - 一度だけライフ 1 で復活 + FEVER ゲージ回復 + スコアボーナス

## テクニカル構成（ざっくり）

- TypeScript
- HTML Canvas 2D API での描画
- face-api.js をベースとした表情推定（`face.ts` / `camera.ts` まわり）
- ドメインロジックの分割：
  - `game.ts` … ゲームループ・状態更新・表情入力との結合
  - `draw.ts` … 描画専用（背景・プレイヤー・爆弾・スター・UIなど）
  - `gameTypes.ts` … Bomb / Star / Wave / Mission / PublicState などの型定義
  - `cosmetics.ts` … コイン・スキン・ガチャなど、コスメティクス系ドメイン
  - `dom.ts` … DOM 初期化や Canvas セットアップ
  - `face.ts` / `camera.ts` … カメラ・顔認識まわり
  - `main.ts` … エントリーポイント

クリーンアーキテクチャを意識して：

- 「表情 → Expression の列挙型（`Expression = "happy" | "angry" | ...`）」という入力
- 「ゲーム状態更新（物理・スコア・FEVER・トレンド・ガチャなど）」
- 「描画（純粋に Canvas に描くだけ）」

をできるだけ分離しています。

## セットアップ & 実行方法

> ※ 以下は一般的な Node.js + npm プロジェクトを想定した例です。  
> プロジェクトの実際の `package.json` に合わせてコマンド名は調整してください。

### 1. リポジトリをクローン

```bash
git clone https://github.com/catlover-bot/emotion-game.git
cd emotion-game
````

### 2. 依存関係のインストール

```bash
npm install
# または
# pnpm install
# yarn install
```

### 3. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで表示される URL（例: `http://localhost:5173` など）を開いて、
カメラ利用の許可を出すとゲームが起動します。

### 4. ビルド

```bash
npm run build
```

ビルド成果物は `dist/` などに出力されます（ツールによって異なります）。

## セキュリティ・プライバシーについて（予定方針）

* カメラ映像は **ブラウザ内でのみ処理**され、サーバーには送信しない設計を想定
* 表情推定のための特徴量も、外部には送信せず端末内で完結
* 将来 App Store に申請する際には、

  * プライバシーポリシー
  * カメラ利用の目的（NSCameraUsageDescription）
    を明示する予定です
